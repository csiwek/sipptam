4 # Summary of functionality
5 
6 This package provides a lot of miscellaneous things that probably should have been included in python's built-in
7 library but weren't, including:
8 
9 - Timing execution of a function with a decorator
10 - An abstract class method
11 - A way to get the argument names and values of a function without using **kwargs or *args
12 - An improved ArgParser for parsing command line arguments
13 - A poor man's debug tracer when a better tracer isn't available (for example, when running a python script over ssh)
14 - A way to describe the argument and return types of functions, kind of similar to a statically typed language
15 - A module to get details about a python package (Is it built in? In what file is it? etc.)
16 - LIFO/Stack implementation
17 - Easy event logging
18 - Terminal/user interaction improvements

44 ## Getting a function's arguments
45 
46 Python provides `**kwargs` and `*args` to get a dictionary or list of a function's arguments, but this makes it hard
47 for IDEs and documentation generators to determine all possible arguments to the function. As an alternative to
48 `**kwargs` or `*args`, you can use the `getArgs()` method:
49 
50  from Lang import getArgs
51  def myFunc(arg1, arg2):
52  allArgs = getArgs()
53 
54 This method uses python's built in `inspect` module to go up the stack and inspect arguments.
55 
56 In the case above, `allArgs` will be a list of values, similar to as if `*args` was used. To get a dictionary of
57 argument names and values instead, similar to `**kwargs`, use `getArgs(useKwargFormat=True)`.
58 
59 Note that `cls` and `self` are automatically ignored for class methods and instance methods.

61 ## An improved ArgParser
62 
63 In addition to the plethora of features in python's built in `ArgParser`, a few more are added in here:
64 
65 - Improved help formatting, similar to `man`
66 - 3 way boolean (`True`, `False`, `None`)
67 - Required named parameters - the built in ArgParser only supports required positional arguments and optional named parameters
68 
69 The new `ArgParser` uses the same interface as the old one, so see the built in `ArgParser` documentation.
70 
71 Example:
72 
73  from Lang.ArgParser import ArgParser
74  parser = ArgParser(argument_default=None, add_help=True, description="Adds a user to a linux machine")
75  parser.add_argument("username")
76  parser.add_argument("-p", "--password", required=False, help="Prompt for password if this is not given")
77  parser.add_argument("-H", "--create-home", type=Bool3Way, required=True,
78  help="Controls home directory creation for user. None uses the default behavior which varies between machines.")
79  args = parser.parse_args()

166 ## Event logging
167 
168 Python has decent built in logging, but it doesn't follow standard object-oriented concepts where methods represent
169 actions, so the API is not ideal for recording different events in a heavily event based system, as there would need
170 to be a special, separate call to the logging API for every event. The logging API in `Lang.Logging` fixes this.
171 
172 ### Example using StdoutLogger
173 
174 This is a very simplistic example of logging to stdout:
175 
176  from Lang.Logging import Logging, StdoutLogger
177  log = Logging(StdoutLogger)
178  log.notifyMyEvent("details", "in", "arguments", "here")
179 
180 ### Example using a custom logger and/or multiple loggers
181 
182 When using multiple loggers, the function you call on the `Logging` instance will be called on every logger.
183 
184  from Lang.Logging import Logging, LoggerAbstract, StdoutLogger
185  class MyFileLog(LoggerAbstract):
186  def __init__(self, filePath):
187  super(MyLogger, self).__init__()
188  self._filePath = filePath
189  def notifyFolderCheck(self, folder):
190  with open(self._filePath, "a") as file_:
191  file_.write("Checking folder: " + folder)
192 
193  log = Logging((StdoutLogger, MyFileLog))
194  log.notifyFolderCheck("folder/path/here")
195 
196 ### Logging uncaught exceptions
197 
198 When an uncaught exception happens, a special `notifyException` method will be called on each of your loggers
199 automatically if it exists, with the exception instance and a traceback instance as parameters. What you do with
200 these parameters is up to you, but here is an example:
201 
202  def notifyException(self, exceptionInstance, tracebackInstance):
203  import traceback
204  tracebackStr = "".join(traceback.format_tb(tracebackInstance))
205  exceptionStr = str(exceptionInstance)
206  print(tracebackStr)
207  print()
208  print(exceptionStr)

220 ### Using formatted text on the terminal
221 
222  from Lang.Terminal import FormattedText
223 
224 `FormattedText` returns a subclass of the built in python `str` type. It adds terminal formatting codes when
225 the `str` function is called on it:
226 
227  boldStr = str(FormattedText("I'm in bold", bold=True))
228 
229 `boldStr` will contain some funny looking characters which enable formatting, but these characters do not influence
230 any other function of the string. For example:
231 
232  char = boldStr[2]
233 
234 will store `m` into `char`. Similarly, `len` and other functions will act the same as if the string were a direct
235 instantiation of `str`.
236 
237 Many more text attributes are available, including coloring - see the source for more information.
238 
239 ### Tables
240 
241 Printing tabular data to the terminal is very common. This implementation of a table has a couple extras with it.
242 Here is an example:
243 
244  from Terminal import Table
245  table = Table()
246  table.setColHeaders(("First Name", "Last Name Initial", "Number")) # optional
247  table.setColMaxLens([None] * 3) # enables automatic column sizing for the 3 columns
248  table.addRow(["Jesse", "C", str(1234)]) # explicitly convert all elements to strings
249  table.printLive() # prints all rows (and headers) that haven't been printed before
250 
251 If another row is added to the table, another call to `printLive()` will only print only that new row.
